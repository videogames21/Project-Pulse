# Team Workflow

## Branching

- main = stable
- feature/use-case-name

---

## Pull Requests

- Required for all changes
- Must include:
  - Description
  - Related use case
  - Test evidence

---

## Code Review

- Check:
  - Requirements alignment
  - Architecture compliance
  - Code clarity

---

## Rules

- No direct push to main
- No major architectural change without discussion
- Keep communication clear
